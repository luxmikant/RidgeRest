Objective

Build a basic web application where employees can apply for leave and employers can approve or reject those requests.

Important Notes

This is a paid internship with the possibility of conversion to a full-time role (FTE) based on performance.

The internship is work-from-office, and candidates must be available to work from our Bangalore office five days a week.

Candidates are required to bring their own laptop for work. A minimum of 16 GB RAM is recommended.

Core Features

Employee users should be able to sign up, log in, apply for leave by providing leave type, start date, end date and reason, and view the status of their leave applications (Pending, Approved or Rejected).

Employer users should be able to sign up, log in, view all employee leave requests, and approve or reject those requests.

Tech Stack (Mandatory)

The frontend should be built using Vue.js and Tailwind CSS.

The backend can be implemented using either Node.js or Python and should expose REST APIs.

The database should use MongoDB Atlas.

Deployment Requirement

The application must be hosted on any free cloud platform so that it can be accessed through a public URL.

Examples include Amazon Web Services Free Tier, Google Cloud Free Tier, Microsoft Azure Free Tier, or any other free cloud hosting service.

Please provide a working live URL of the deployed application.

Basic Requirements

The application should have a simple and functional user interface. Design aesthetics are not the focus.

The code should be clean and readable with a proper folder structure.

Environment variables should be used for sensitive information such as database connection strings.

Basic error handling should be implemented.

Use of AI Tools

You are allowed to use AI tools such as ChatGPT, Claude, or GitHub Copilot to assist with coding.

However, you should have a basic understanding of the code you submit and be able to explain the code structure and flow. You should also be able to walk through how the application works and explain how the deployment is set up, including the deployment topology.

You are not expected to memorize syntax, but you should understand the logic and architecture of your solution.

Optional (Bonus Points)

Additional credit may be given if the solution includes JWT-based authentication, role-based access control between Employee and Employer users, basic input validation such as required fields and date validation, and a well-written README explaining setup and deployment.

Submission

Please submit the GitHub repository link along with the live application URL. The repository should include a README containing project setup steps, API endpoint details, and deployment instructions.

Expected Effort and Timeline

The expected effort for this assignment is approximately 4 to 12 hours. The use of AI coding assistants is allowed.

The submission timeline is within 48 hours of receiving the assignment. Earlier submissions will have a better chance of being considered, as there is only one position available
